Packages, import and visability

  • A program is constructed as a package
  • Many .go files can belong to one package
  • Package Declaration - The package to which the code-file belongs must be indicated on the first line
  • Each Go application conatins one package called main
  • So, If I have this right...
    • Many statements can be in a source-code file
    • Many source code files can be in a package
    • Many packages can be in an application
    • The application package with the standalone executable is called "main"package

Jump Section

Standard Library

  • Ready to use packages that are part of the Go installation
  • A Go program is created by linking together a set of packages through the import declaration
    • example: import "fmt" tells go that the program needs elements from the package fmt.
    • Import "loads" the public declarations from the compiled package
