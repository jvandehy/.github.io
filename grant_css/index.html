Today I learned about "The Cascade"

July 14, 2018

Fundamentally, CSS is About Declaring Rules

  • We declare rules about what we want to happen under certain conditions
  • Browser takes rules, figures out which apply where
  • Browser renders page

A Part of CSS Development Comes Down to Writing Rules in Such A Way That They're Predictable

  • small examples are straightforward
  • code becomes complex quickly

Predicting how rules behave requires and understanding of the cascade

  • What happens when two rules provide conflicting information
  • How exactly does the browser make sense of your rules?

Example: Basic page header

  1. <!doctype html>
  2. <head>
  3.     <link href="styles.css" rel="stylesheet" type="text/css"/>
  4. </head>
  5. <body>
  6.     <header class="page-header">
  7.         <h1 id="page-title" class ="title">Wombat Coffee Roasters</h1>
  8.         <nav>
  9.             <ul id="main-nav" class="nav">
  10.                 <li><a href="/">Home</a></li>
  11.             </ul>
  12.         </nav>
  13.     </header>
  14. </body>
